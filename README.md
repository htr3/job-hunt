# Job Hunt AI Agent

Python CLI + (upcoming) web dashboard that scrapes jobs from multiple platforms, scores them with an LLM, tracks them in SQLite, and optionally auto-applies.

> **Phase 1 MVP** — this repo currently ships the orchestrator, SQLite store, base scraper contract, and a working **RemoteOK** scraper. Additional platforms, AI matching, the Flask/Streamlit dashboards, and auto-apply land in follow-up phases.

## Quick Start — Run with Docker (recommended for most users)

If you just want to **use** the tool, this is the fastest path. No Python, no
Chrome install, no git clone needed.

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running

### Three commands

```bash
# 1. Download the compose file and the secrets template
curl -O https://raw.githubusercontent.com/<owner>/hunt/main/docker-compose.user.yml
curl -O https://raw.githubusercontent.com/<owner>/hunt/main/.env.example
mv .env.example .env

# 2. Open .env in any text editor and fill in the secrets you want.
#    All fields are optional — leave blank if you don't use that platform.

# 3. Start the app
docker compose -f docker-compose.user.yml up -d
```

Open **http://localhost:5050** in your browser. That's it.

### Updating to a new release

```bash
docker compose -f docker-compose.user.yml pull
docker compose -f docker-compose.user.yml up -d
```

Your job database, reports, and saved Chrome login persist across upgrades.

### Stopping it

```bash
docker compose -f docker-compose.user.yml down
```

Add `-v` to also wipe the database and Chrome profile (full reset).

## Quick Start — Run from source (developers / contributors)

```bash
# 1. Install dependencies (Python 3.11+ recommended)
pip install -r requirements.txt

# 2. Run the interactive setup wizard — it writes config.yaml + .env for you
python job_hunter.py --setup

# 3. Smoke test against RemoteOK (public API, no login needed)
python job_hunter.py --platforms remoteok --no-apply
```

On Windows you can also just double-click **`run_job_hunter.bat`** — it activates `.venv` if present, launches the wizard on first run, and then starts the agent.

## What the setup wizard collects

The wizard (`python job_hunter.py --setup`) walks you through:

- **Profile** — name, email, phone, city, experience years, current role, resume path
- **Search** — target job titles, key skills, excluded keywords
- **Location** — preferred cities, remote-only toggle, relocation willingness
- **Filters** — experience / salary ranges, job type, posted-within-days
- **Platforms** — enable/disable each of 8 sources (LinkedIn, Naukri, Indeed, Glassdoor, Instahyre, RemoteOK, Foundit, Wellfound)
- **Secrets** — OpenAI key, Naukri login, LinkedIn `li_at` cookie, Telegram/Slack tokens (hidden input)
- **AI matching** — enable, choose model, set minimum match score
- **Auto-apply** — enable, daily limit, headless toggle, screening answers
- **Notifications** — desktop / email / telegram / slack / whatsapp
- **Output** — results dir, report format, max results per platform

All **secrets land in `.env`** (git-ignored). `config.yaml` references them as `${OPENAI_API_KEY}`, `${NAUKRI_PASSWORD}`, etc., and they're expanded at load time.

## CLI flags

| Flag | Description |
| --- | --- |
| `--setup` | Run the interactive wizard, write `config.yaml` + `.env`, then exit |
| `--config PATH` | Use a non-default config file (default: `./config.yaml`) |
| `--platforms NAME [NAME...]` | Override enabled platforms for this run |
| `--new-only` | Only report jobs whose URL wasn't in the DB before this run |
| `--no-apply` | Skip the auto-apply stage |
| `--web` | Launch the Flask dashboard on `http://127.0.0.1:5000` after the run |
| `--verbose` | DEBUG-level logging to console and `logs/job_hunter.log` |

## Data model

`results/jobs.db` (SQLite, WAL journal, `busy_timeout=30s`) has two tables:

- **`jobs`** — keyed by unique `url`. Fields: `title, company, location, platform, description, salary, experience, job_type, posted_date, skills, match_score, status, first_seen_at, last_seen_at, applied_at, notes`. Indexed on `url`, `status`, `platform`, `first_seen_at`.
- **`run_history`** — one row per run with `platforms, total_scraped, new_jobs, duration_seconds`.

Valid statuses: `new`, `applied`, `interview`, `rejected`, `offer`, `saved`, `hidden`.

## Engineering invariants

- **cwd-safe DB path** — `jobs.db` is always resolved relative to the project root, not the current working directory, so CLI / Flask / Streamlit all hit the same file.
- **Graceful Ctrl+C** — a shared `threading.Event` is polled by every scraper loop; partial results are saved incrementally.
- **Dead-driver detection** — if a Chrome window is closed mid-run, the scraper logs a single warning and aborts its own work without crashing the others.
- **No secrets in commits** — `.gitignore` excludes `.env`, `__pycache__/`, `results/*.db*`, `reports/`, `tailored_resumes/`.
- **Config-driven** — no hardcoded titles, cities, or credentials anywhere in source.

## Dashboards

**Flask (interactive — spawn runs, edit jobs):**

```bash
python web_app.py --port 5000
# or, after a CLI run:
python job_hunter.py --platforms remoteok --no-apply --web
```

Then open http://127.0.0.1:5000. The UI has five tabs:
- **Run** — pick platforms + flags, live-tail the agent log
- **Jobs** — filter by status/platform/query, inline status dropdown + notes + Save
- **Kanban** — columns by status
- **Config** — read/write `config.yaml` as YAML (a `.yaml.bak` backup is kept)
- **Runs** — recent `run_history` rows

**Streamlit (read-only status board):**

```bash
streamlit run dashboard.py
```

Charts by platform/status, filter sidebar, kanban, recent runs.

## Smoke test checklist

1. `pip install -r requirements.txt`
2. `python job_hunter.py --setup` — walk through the prompts
3. `python job_hunter.py --platforms remoteok --no-apply` — should scrape RemoteOK and populate `results/jobs.db`
4. Re-run with `--new-only` — should report zero new jobs (confirms dedupe)
5. `python web_app.py --port 5050` then `GET http://127.0.0.1:5050/api/stats` returns counts; `POST /api/run` spawns a run you can tail via `/api/logs/<task_id>`.

## Phase 4 modules in detail

Each stage is gated by its own `enabled` flag in `config.yaml`. Stages run in this order after scraping/scoring and each is wrapped in try/except so one failure never blocks the others:

| Stage | Config block | Notes |
| --- | --- | --- |
| Exporters | `export.json` / `export.google_sheets` / `export.notion` | JSON always works; Sheets+Notion are optional imports. |
| Gap analysis | `gap_analysis.enabled` | No LLM needed — pure keyword diff against `search.skills`. |
| Resume tailor | `resume_tailoring.enabled` + `top_n` | Needs `OPENAI_API_KEY`. Writes `tailored_resumes/*.md`. |
| Interview prep | `interview_prep.enabled` + `top_n` | Needs `OPENAI_API_KEY`. Writes `interview_prep/*.md`. |
| LinkedIn network | `linkedin_network.enabled` + `top_n` | Needs `LI_AT_COOKIE`. `auto_connect: true` is intentionally a no-op (accounts get restricted). |
| Auto-apply | `auto_apply.enabled` + `daily_limit` + `rate_limit.naukri` | Needs `NAUKRI_EMAIL` / `NAUKRI_PASSWORD`. Only runs platforms in `auto_apply.platforms` (default: `naukri`). Skipped when `--no-apply` is passed. Jobs whose Apply button hands off to an external company portal are recorded as `skipped_external` (not `applied`) so the count stays honest. |
| Naukri source mode | `platforms.naukri.mode` | `search` (default, anonymous keyword search) or `recommended` (logs in, scrapes `/mnjuser/recommendedjobs`). Recommended jobs have a much higher Easy-Apply rate, so auto-apply succeeds far more often. |
| Notifier | `notifications.{desktop,email,telegram,slack,whatsapp}` | Env vars: `SMTP_HOST/USER/PASSWORD/TO`, `TELEGRAM_BOT_TOKEN/CHAT_ID`, `SLACK_WEBHOOK_URL`, `WHATSAPP_PHONE/APIKEY`. Every channel fails independently. |

### Scheduler

```bash
python scheduler.py            # fire every day at config.scheduler.run_time (HH:MM, local tz)
python scheduler.py --now      # run once immediately and exit
python scheduler.py --at 09:15 # override the daily time
python scheduler.py --no-apply --platforms remoteok  # flags forwarded to each run
```
